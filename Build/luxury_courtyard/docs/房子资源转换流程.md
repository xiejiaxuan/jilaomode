# 房子资源转换流程（DST）

本流程用于把单张 PNG 图片转换成当前 Mod 可直接识别的房屋门资源：

```text
Build/luxury_courtyard/anim/jm_house_door.zip
```

## 输入文件

- 原图：`C:\Users\Administrator\Pictures\AI\jl\smallhouse.png`
- Klei 工具目录：`E:\Program Files (x86)\Steam\steamapps\common\Don't Starve Mod Tools`

## 当前项目约定

`scripts/prefabs/jm_house_door.lua` 会自动检测：

```text
anim/jm_house_door.zip
```

检测到后使用：

- Bank：`jm_house_door`
- Build：`jm_house_door`
- 动画：`close`

门被使用时还会播放 `open`，然后回到 `close`，所以资源包里建议同时提供：

- `close`
- `open`

即使只是静态图，也要生成这两个动画名。

## 已采用的转换方式

1. 将原始 1024x1024 PNG 缩放为 256x256 中间图：

```text
Build/luxury_courtyard/_asset_work/jm_house_door/house/house-0.png
```

2. 生成中间文件：

```text
Build/luxury_courtyard/_asset_work/jm_house_door/build.xml
Build/luxury_courtyard/_asset_work/jm_house_door/animation.xml
```

3. 将中间文件压缩为编译源包：

```text
Build/luxury_courtyard/_asset_work/jm_house_door.zip
```

4. 调用 Klei 的 `buildanimation.py`：

```powershell
& "E:\Program Files (x86)\Steam\steamapps\common\Don't Starve Mod Tools\mod_tools\buildtools\windows\Python27\python.exe" `
  "E:\Program Files (x86)\Steam\steamapps\common\Don't Starve Mod Tools\mod_tools\tools\scripts\buildanimation.py" `
  "D:\jilaomode-main\Build\luxury_courtyard\_asset_work\jm_house_door.zip" `
  --outputdir=. --force --ignoreexceptions
```

5. 输出结果：

```text
Build/luxury_courtyard/anim/jm_house_door.zip
```

生成的 zip 内应包含：

```text
anim.bin
build.bin
atlas-0.tex
```

## 备注

- 单张 PNG 不能直接被 DST 当作实体动画资源使用，必须打包成 `anim/*.zip`。
- 当前做法是“静态动画”：`open` 和 `close` 都显示同一张房子图。
- 如果后续要做开门动画，可以增加多张帧图，并在 `animation.xml` 中加入多帧。
- 若游戏内尺寸不合适，优先重新生成 128x128、192x192、256x256 或 384x384 中间图后再打包。
