# 资源接入模板（DST）

这个模板的目标是：**你把资源放到约定路径后，代码自动切换到自定义资源**。  
如果资源还没准备好，会自动回退到原版占位资源，不影响功能测试。

## 一、目录模板

请在 Mod 根目录准备以下结构：

```text
anim/
  jm_house_door.zip
  jm_deco_plant.zip

images/
  inventoryimages/
    jm_deco_plant_kit.tex
    jm_deco_plant_kit.xml
```

## 二、资源命名约定

- 门动画包：`anim/jm_house_door.zip`
  - Bank/Build 建议都叫 `jm_house_door`
  - 至少包含动画 `close`（目前放置预览和实体都用到）
- 盆栽动画包：`anim/jm_deco_plant.zip`
  - Bank/Build 建议都叫 `jm_deco_plant`
  - 至少包含动画 `idle`
- 背包图标：`images/inventoryimages/jm_deco_plant_kit.tex + .xml`
  - 图集内 sprite 名称建议为 `jm_deco_plant_kit.tex`

## 三、当前代码已做的自动接入

你现在不需要再改 Lua，代码已支持：

1. 若检测到 `anim/jm_house_door.zip`，门自动使用自定义动画。  
2. 若检测到 `anim/jm_deco_plant.zip`，家具和家具套件自动使用自定义动画。  
3. 若检测到 `images/inventoryimages/jm_deco_plant_kit.xml`，背包图标自动切换并注册 atlas。  
4. 任一资源缺失时，自动回退到原版 `treasure_chest / pottedfern` 占位资源。

## 四、如何制作与导出资源

常见流程（美术管线）：

1. 在 Spriter/Photoshop 等工具产出序列帧与符号。  
2. 用 Klei 的导出工具打包为 `anim/*.zip`。  
3. 图标用 `texconv` / Klei 工具链生成 `.tex`，并生成对应 `.xml` 图集。  
4. 放入上述目录，重启世界测试。

## 五、接入检查清单

- 门放置后外观是否是你的自定义门（非箱子外观）
- 盆栽 kit 的背包图标是否已变为自定义图标
- 放置出的盆栽是否为自定义模型且有 `idle` 动画
- 联机情况下其他玩家是否也能看到同样资源

## 六、常见问题

- 看不到自定义动画：通常是 Bank/Build 名称与 Lua 不一致。  
- 图标不显示：通常是 `.xml` 中 sprite 名和 `imagename` 不匹配。  
- 只有自己看得到：确认客户端和服务器都安装了同版本 Mod 资源。
